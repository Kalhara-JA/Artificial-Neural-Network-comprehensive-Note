01

Introduction to Artificial Intelligence

What is Artificial Intelligence?

Artificial Intelligence (AI) is the science and engineering of building intelligent machines — systems that can perceive their environment, reason about it, and take actions to achieve goals.

Intelligence is not an absolute property. It is comparative and contextual. What counts as "intelligent" depends on the environment, the observer, and the task. There is no single universal definition because different fields (neuroscience, psychology, computer science) each propose different criteria.

💡 Key Insight

Think of intelligence as a spectrum, not a switch. A chess engine is "intelligent" at chess but completely helpless at driving a car. Context always defines intelligence.

The Cycle of Intelligent Behavior

Every intelligent system — biological or artificial — operates in a three-step cycle. Understanding this cycle is fundamental to understanding how AI works:

Figure 1.1 — The Cycle of Intelligent Behavior
  1. Perceive and Understand — Extract features from the world. This is inherently biased by the agent's goals, desires, and background knowledge. A cat and a human looking at the same scene extract very different features.
  2. Inferencing on Knowledge — The extracted features interact with a knowledge model. The model reasons about what the features mean and what should happen next, guided by the agent's objective.
  3. Plan and Execute — Based on the inference, the agent forms an action plan and acts on the environment. The consequence of this action produces new perceptions — closing the loop.

Knowledge Representation

AI systems need to store and use knowledge. There are two main approaches:

Rule-Based (Symbolic)

Knowledge encoded as explicit rules and facts: "IF temperature > 38°C THEN patient has fever". Easy to inspect and explain, but rigid — you must manually write every rule. Hard to scale.

Data-Oriented (Statistical / ML)

Knowledge extracted automatically from data through experience. The model learns rules from examples rather than being told them. Scales well, but can be a "black box".

Ethical Guardrails and Cost Functions

Intelligent machines need guardrails — constraints that prevent them from pursuing objectives in harmful or undesirable ways. In ML, these are implemented mathematically as cost/loss functions.

A cost function penalizes the model for making bad decisions. It is the mathematical "conscience" of the AI. The model is trained to minimize this cost, which guides its behavior toward desirable outcomes.

⚠️ Important Concept

Goals and desires drive the knowledge model. Without proper cost functions and guardrails, an AI could pursue its objective in ways that are technically correct but ethically harmful. This is why "Responsible AI" is a real engineering concern, not just a philosophical one.

Major Areas of AI

Machine Learning

Building knowledge models from data. The core of modern AI. Enables systems to improve with experience.

Natural Language Processing

Understanding and generating human language. Powers chatbots, translation, sentiment analysis.

Computer Vision

Perceiving and interpreting visual information. Powers facial recognition, self-driving cars, medical imaging.

Multi-Modal Models

Combining text, image, audio in one model. Modern trend: GPT-4V, Gemini, Claude can all handle multiple modalities.

Transformers

The dominant modern architecture. Use attention mechanisms. Power almost all state-of-the-art models in NLP and vision.

Reinforcement Learning

Learning from rewards and penalties. Powers game-playing AI (AlphaGo), robotics, recommendation systems.

Fundamentals of Machine Learning

What is Learning?

Learning is the process of improving performance over time through interaction with an environment. A system learns by:

  • Gathering experiences from data or environmental feedback
  • Extracting relevant information from those experiences
  • Using that information to update its internal knowledge model (the parameters θ)
  • Using feedback (which can be immediate or delayed) as a training signal

A learning algorithm is the specific process that allows the model to continuously improve by adjusting its parameters based on experience.

The Three Phases of a Machine Learning Model

Every ML model goes through distinct phases. Understanding which phase you're in is crucial because what you're allowed to do is different in each:

Figure 2.1 — Three Phases of an ML Model
⚠️ Critical Rule

Weight updates ONLY happen during training. During testing and inference, the weights θ are completely frozen. If you update weights on test data, you are "cheating" — the model will appear to perform well but will fail on truly new data.

03

Types of Learning

Machine learning algorithms are classified by how they receive feedback during training. Each type is suited to different kinds of problems and data.

1. Supervised Learning

You provide the model with labeled data — pairs of input X and the correct output Y. The model learns a mapping from X → Y by minimizing the difference between its predictions and the true labels.

Figure 3.1 — Supervised Learning Loop

Regression

Target Y is a continuous number. The model predicts a value on a continuous scale. Examples: predicting house prices, forecasting temperature, estimating salary. Error is computed arithmetically (e.g., predicted − actual).

Classification

Target Y is a discrete category. The model must assign inputs to one of several classes. Examples: spam/not-spam, digit recognition (0–9), disease diagnosis. The model learns decision boundaries to separate classes.

2. Unsupervised Learning

No labels are provided — only raw input data X. The model must discover structure on its own. The most common task is clustering: grouping similar inputs together.

The model uses mathematical tools like Euclidean distance or cosine similarity to determine how "close" inputs are to each other. Similar inputs end up in the same cluster.

💡 Analogy

Imagine sorting a pile of mixed fruit with no labels. You group them by shape, color, and size. You discover "apples", "oranges", and "bananas" — but you named these clusters yourself. That's unsupervised learning.

3. Reinforcement Learning

An agent learns by interacting with a dynamic environment. It takes actions and receives rewards (positive or negative). The goal is to learn a policy that maximizes cumulative reward over time. Rewards can be delayed — an action taken now may only show consequences many steps later.

Figure 3.2 — Reinforcement Learning Agent-Environment Loop

4. Self-Supervised Learning

A clever middle ground. The model uses the structure of raw data itself to generate supervision signals — no manual labeling needed. It operates in two stages:

1
Pretext Task (Pre-training)

The model learns from the inherent structure of data. Example: given a sentence, predict the next word. This requires understanding grammar, meaning, and context — all learned without any human labels. The model learns to understand the domain.

2
Downstream Task (Fine-tuning)

Now apply the pre-trained model to a specific task (e.g., sentiment analysis, summarisation). Only a small amount of labeled data is needed because the model already "understands" the language from step 1.

💡 Real-World Example

GPT models are trained with self-supervised learning. GPT-3 was pre-trained on billions of words by predicting the next token (pretext). Then fine-tuned on specific tasks with human feedback (downstream). This is why GPT can do summarisation, Q&A, coding, etc. — it learned the "language of the world" first.

ANN Data Preprocessing

Neural networks are hungry for data — they consume enormous amounts. This changes how we handle data compared to classical ML. The preprocessing steps below are specific to neural networks and understanding why each step matters is as important as knowing what to do.

Data Cleaning

In classical ML with small datasets, missing values are imputed (filled in) carefully. In ANN with large datasets, the simpler approach is usually just to drop the affected rows. With millions of samples, losing a few hundred rows is negligible and avoids introducing imputation bias.

Encoding Categorical Variables

Neural networks only understand numbers. Every categorical variable (e.g., "color", "gender", "city") must be converted to numbers. But how you encode matters enormously:

Method How It Works Example Use / Danger
One-Hot Encoding Create a binary column for each category. Only one column is 1 at a time. Red → [1,0,0]
Blue → [0,1,0]
Green → [0,0,1]		 ✅ No false order introduced
⚠️ Dimensionality explosion with many categories
Integer Encoding Assign an integer to each category. Low=1, Medium=2, High=3 ✅ Good for ordinal (ordered) variables
❌ Introduces false numeric relationships for nominal data
Binary / Oppositional Use 1/0 or +1/−1 pairs Male=1, Female=0
or Male=+1, Female=−1		 ⚠️ Using 0 silences a neuron; use +1/−1 (excitation/inhibition)
⚠️ The Zero Problem in Encoding

If you encode a class as 0, the neuron receiving that input gets no signal at all for that class — it can't learn to distinguish it. Using −1 and +1 instead sends an active inhibitory signal (−1) vs. an excitatory signal (+1), giving the neuron information in both directions.

Feature Scaling

Should you scale inputs to a neural network? It depends — but here's the nuance specific to ANNs:

Standardization (z-score)

Transforms to mean=0, std=1. This can push values negative. Negative inputs can cause some activation functions (like ReLU) to output zero — effectively "depressing" or silencing neurons. Use with caution.

x' = (x − μ) / σ

Min-Max Scaling

Compresses values to [0, 1]. Keeps all values non-negative. Generally safer for ANNs. But if you need inhibition signals, use [−1, +1] range instead.

x' = (x − min) / (max − min)

Variable Discretization (Binning)

This is a mandatory step for continuous variables with large ranges in ANNs. Instead of feeding the raw value (e.g., age = 47), you convert it to a discrete bin (e.g., age_40_50 = 1).

Why does this matter for ANNs? Neural networks form "memory paths" — specific weight patterns that activate for specific inputs. If age ranges from 0 to 100 with decimal precision, there are effectively infinite possible values. Binning reduces this to a small, manageable set of categories that the network can reliably learn to distinguish.

💡 Think of it like this

Would you rather teach a student to recognize people by their exact height in millimeters (1,754mm, 1,755mm, 1,756mm...) or by height categories (short, medium, tall)? The categories are much easier to learn and generalize from.

Class Imbalance Handling

In many real-world datasets, one class heavily outnumbers another (e.g., 99% normal transactions, 1% fraud). This creates a dangerous problem:

⚠️ The Accuracy Trap

A model that always predicts the majority class will achieve 99% accuracy on fraud detection — but it has learned absolutely nothing useful. It will miss every single fraud case. Never use accuracy alone on imbalanced data.

TechniqueApproachWhen to Use
SMOTE (Synthetic Minority Oversampling) Generates new synthetic minority-class samples by interpolating between existing minority points, especially near the decision boundary. When minority class has very few samples. Most widely used.
Random Oversampling Duplicates existing minority samples at random. Quick fix; risk of overfitting on repeated samples.
NearMiss (Undersampling) Removes majority-class samples that are "far" from the decision boundary, keeping only the most informative ones. When you have a very large majority class and can afford to lose data.
Class Weights Penalise misclassifications of the minority class more heavily in the loss function. Simple and effective. No data modification needed.

The Artificial Neuron — From Biology to Math

Biological Inspiration

Artificial neural networks were originally inspired by the structure of the human brain. Understanding the biological analogy helps you remember why the math is structured the way it is.

Figure 5.1 — Biological Neuron vs. Artificial Neuron
Biological ComponentArtificial EquivalentRole
DendritesInput connections (x₁, x₂, ...)Receive signals from other neurons
Synapse strengthWeights (w₁, w₂, ...)Amplify or diminish each signal
Cell bodySummation Σ(xi·wi) + bIntegrate all incoming signals
Axon hillock thresholdActivation function f(z)Decide whether to "fire"
Axon outputOutput ŷSend result to next neurons
NeuroplasticityWeight updates (learning)Strengthen/weaken connections over time

The Neuron Computation — Step by Step

Every neuron does exactly two things:

z = w₁x₁ + w₂x₂ + ... + wₙxₙ + b = Σ(wᵢ · xᵢ) + b Step 1: Weighted sum of inputs plus bias
output = f(z) Step 2: Pass through activation function
📐 Worked Example — Single Neuron Computation
Given: x = [1.0, 0.5, 0.8], w = [0.4, 0.6, -0.3], b = 0.1
Step 1: z = (1.0 × 0.4) + (0.5 × 0.6) + (0.8 × −0.3) + 0.1 z = 0.40 + 0.30 + (−0.24) + 0.10 z = 0.56
Step 2: output = sigmoid(0.56) = 1 / (1 + e^−0.56) output = 1 / (1 + 0.5712) output = 1 / 1.5712 ≈ 0.6364
Meaning: This neuron outputs 0.636 — interpreted as a 63.6% probability if used in binary classification.

Learning as Memory Path Creation

When a neural network trains, it is creating and strengthening specific pathways through the weight matrix. A particular input pattern activates a specific sequence of neurons — this is the network's "memory" of that pattern.

Over many training examples, the weights adjust so that:

  • Input pattern A → strong activation on path A → output class A
  • Input pattern B → strong activation on path B → output class B
  • Similar inputs → similar activation patterns (generalization)
  • The bias term allows each neuron's activation threshold to shift, making the network more flexible
💡 The Bias Term Explained

Think of the bias like the constant c in y = mx + c. Without it, every line would have to pass through the origin (0,0), which is very limiting. The bias lets the model shift its decision boundary anywhere in space, not just through zero. This is why including bias always improves flexibility.

ANN Performance Evaluation

How do you know if your neural network is actually working? You need the right metrics for the right problem. Using the wrong metric can make a terrible model look great.

Regression Metrics

MetricFormulaCharacteristicsWhen to Prefer
MAE — Mean Absolute Error mean(|y − ŷ|) Same units as y. All errors weighted equally. Robust to outliers. When all errors matter equally. Easy to explain to non-technical stakeholders.
MSE — Mean Squared Error mean((y − ŷ)²) Units are squared. Large errors penalised much more than small ones. Sensitive to outliers. When large errors are especially costly (e.g., medical dosage prediction).
RMSE — Root Mean Squared Error √MSE Same units as y. Combines MSE's outlier sensitivity with interpretable scale. Most commonly reported metric. Good default for regression.
R² — R-squared 1 − SS_res/SS_tot Scale-independent: ranges from −∞ to 1.0. Not affected by units of y. When comparing models on different scales. 1.0 = perfect fit; 0 = no better than the mean.
📐 Worked Regression Metrics Example

True: y = [3, 5, 2, 8]  |  Predicted: ŷ = [2.5, 5.2, 1.8, 7.0]

Errors: [0.5, 0.2, 0.2, 1.0]

MAE = (0.5 + 0.2 + 0.2 + 1.0) / 4 = 1.9/4 = 0.475

MSE = (0.25 + 0.04 + 0.04 + 1.00) / 4 = 1.33/4 = 0.3325

RMSE = √0.3325 ≈ 0.577

Training Mechanics

The IID Assumption

For gradient descent to work properly, training data must be Independent and Identically Distributed (IID):

Independent

Each training sample must be statistically independent of the others. No sample should "influence" or be correlated with another. This is achieved by shuffling the dataset before training.

Identically Distributed

All samples must come from the same underlying distribution — same mean and variance. This ensures the model learns a consistent pattern, not different patterns from different data sources mixed together.

Batch Types — How Often Do We Update Weights?

Figure 7.1 — Comparing SGD, Batch GD, and Mini-Batch GD on 12 samples
MethodUpdate FrequencyProsCons
Stochastic GD (SGD) After every single sample Very frequent updates; can escape local minima; good for online learning Very noisy gradients; unstable convergence; slow due to no parallelism
Full Batch GD After entire dataset Stable, accurate gradient estimates; smooth convergence Memory intensive; slow for large datasets; one update per epoch
Mini-Batch GD ✅ After every N samples Best of both worlds; GPU parallelism; good convergence; industry standard Requires tuning batch size (typically 32, 64, 128, 256)

Key Terminology

Iteration

One weight update — processing one mini-batch through forward and backward pass, then updating weights.

Epoch

One full pass through the entire training dataset. If dataset = 1000 samples and batch size = 100, then 1 epoch = 10 iterations.

📐 Epoch vs Iteration Calculation

Dataset = 10,000 samples, Batch size = 128

→ Iterations per epoch = ceil(10,000 / 128) = 79 iterations

→ After 50 epochs = 50 × 79 = 3,950 total weight updates

Underfitting, Overfitting & Early Stopping

This is one of the most important concepts in all of machine learning. Getting this wrong means your model either can't learn anything useful, or it "memorises" the training data and fails on anything new.

Underfitting

Model too simple for the complexity of the data. Like trying to fit a straight line through data shaped like a curve. Both training AND test error are high. The model hasn't learned the pattern.

Fix: Add more neurons/layers, use better features, train longer, use a more expressive activation function.

Overfitting

Model too complex — it "memorises" the training data including its noise. Training error is very low, but test error is high. The model fails on any data it hasn't seen before.

Fix: Reduce neurons/layers, add dropout/regularization, get more data, use early stopping.

Figure 8.1 — Underfitting vs Best Fit vs Overfitting

Early Stopping — The Smart Solution

Instead of guessing the right number of epochs upfront, early stopping monitors the model during training and automatically stops when it detects overfitting beginning to occur.

Data is split into three sets:

  • Training set (70%) — used to update weights
  • Validation set (10–15%) — monitored during training but NOT used to update weights
  • Test set (15–20%) — completely held out; used only for final evaluation
Figure 8.2 — Early Stopping: Stop when validation loss begins to rise

Activation Functions

Without activation functions, a neural network — no matter how many layers deep — is mathematically equivalent to a single linear transformation. Activation functions introduce non-linearity, which is what gives neural networks their power to learn complex patterns.

📌 Why Non-linearity Matters

Real-world data is almost never linearly separable. Language, images, medical signals — all require complex, curved decision boundaries. Activation functions are what allow the network to bend and curve these boundaries to fit the data.

Every Activation Function in Detail

Figure 9.1 — Visualising All Major Activation Functions
FunctionFormulaOutput RangeDerivativeBest For
Step / Threshold 1 if z≥0, else 0 {0, 1} Not differentiable (undefined at 0) SLP output when labels are 0/1. Cannot use with backprop!
Sign Function +1 if z≥0, else −1 {−1, +1} Not differentiable SLP output when labels are +1/−1. Cannot use with backprop!
Sigmoid σ 1 / (1 + e^−z) (0, 1) σ(z) · (1 − σ(z)) Binary classification output layer. Avoid in deep hidden layers (vanishing gradient).
Tanh (e^z − e^−z) / (e^z + e^−z) (−1, 1) 1 − tanh²(z) Hidden layers — zero-centered, better than sigmoid for hidden layers.
ReLU max(0, z) [0, ∞) 1 if z>0, else 0 Hidden layers — fast, efficient. Most common default. Risk: "dying ReLU" if all inputs are negative.
SiLU (Swish) z · σ(z) (≈−0.28, ∞) σ(z) + z·σ(z)(1−σ(z)) Hidden layers — smooth, slightly negative values allowed. Often outperforms ReLU in practice.
Softmax e^zᵢ / Σe^zⱼ (0, 1), sums to 1 More complex (Jacobian) Multi-class classification output layer. Converts logits to probability distribution.

Choosing the Right Activation

📌 Decision Guide

Output layer: Binary classification → Sigmoid  |  Multi-class → Softmax  |  Regression → Linear (none)

Hidden layers: Default → ReLU  |  If dying ReLU problem → SiLU / LeakyReLU  |  Older networks → Tanh

Never use Step or Sign in hidden layers or MLP — they are not differentiable and cannot be used with backpropagation.

⚠️ Vanishing Gradient Problem

Sigmoid and Tanh "squash" their inputs into a small range. Their derivatives are very small (close to 0) in the saturated regions. In deep networks, multiplying many small numbers together during backpropagation makes the gradient vanish — weights in early layers barely update. This is why ReLU became the standard: its derivative is simply 1 for positive inputs, preventing gradient decay.

📐 Sigmoid Derivative Calculation (needed for backprop)
Given: z = 0.8
σ(z): σ(0.8) = 1 / (1 + e^−0.8) = 1 / (1 + 0.449) = 1 / 1.449 ≈ 0.690
σ'(z): σ'(0.8) = σ(0.8) × (1 − σ(0.8)) = 0.690 × (1 − 0.690) = 0.690 × 0.310 ≈ 0.2139
Note: The maximum value of σ'(z) is 0.25 (at z=0). This means every pass through a sigmoid shrinks the gradient by at least 75% — demonstrating the vanishing gradient problem in deep nets.

Single Layer Perceptron (SLP)

The Single Layer Perceptron is the simplest neural network: a single neuron with no hidden layers. It's the "Hello World" of neural networks — essential to understand before moving to deeper architectures.

SLP Architecture

Figure 10.1 — Single Layer Perceptron Architecture

SLP Training Rule (Weight Update)

The SLP uses a simple delta rule for updating weights after each sample:

Wnew = Wold + α × (y − ŷ) × x Where α = learning rate, y = true label, ŷ = predicted output, x = input
bnew = bold + α × (y − ŷ) Bias update: same error signal, no x multiplier
📐 Full SLP Training Example
Setup: x = [1.0, 0.5], w = [0.3, 0.7], b = −0.2, α = 0.1, y = 1 (true label) Activation: sigmoid
Step 1 — Forward: z = (1.0 × 0.3) + (0.5 × 0.7) + (−0.2) z = 0.30 + 0.35 − 0.20 = 0.45
Step 2 — Activate: ŷ = sigmoid(0.45) = 1/(1+e^−0.45) = 1/(1+0.638) = 1/1.638 ≈ 0.611
Step 3 — Error: e = y − ŷ = 1 − 0.611 = 0.389
Step 4 — Update W: ΔW = α × e × x = 0.1 × 0.389 × [1.0, 0.5] = [0.0389, 0.0195] W_new = [0.3 + 0.039, 0.7 + 0.019] = [0.339, 0.719]
Step 5 — Update b: Δb = 0.1 × 0.389 = 0.0389 b_new = −0.2 + 0.039 = −0.161

Decision Boundary of SLP

The SLP creates a linear decision boundary — a straight line in 2D, a flat plane in 3D, or a hyperplane in higher dimensions. Every point on one side gets class 0; every point on the other side gets class 1.

Figure 10.2 — Linear Separability: AND (solvable by SLP) vs XOR (not solvable)
⚠️ Fundamental Limitation of SLP

An SLP can only solve problems where the classes are linearly separable — where a single straight line (or hyperplane) can separate them. XOR is the classic example of a problem that is NOT linearly separable. This limitation motivated the invention of the Multi-Layer Perceptron (MLP) with hidden layers.

Multi-Layer Perceptron (MLP)

The MLP solves the SLP's fundamental limitation by adding hidden layers between the input and output. These hidden layers transform the input into progressively more abstract representations, enabling non-linear decision boundaries.

MLP Architecture

Figure 11.1 — Multi-Layer Perceptron: 4 inputs → 4 hidden → 3 hidden → 2 outputs

Matrix-Based Forward Propagation

In an MLP, each layer's computation is a matrix multiplication followed by bias addition and activation. This is efficiently computed using linear algebra:

📐 Layer-by-Layer Forward Propagation (2-Layer MLP)
Architecture: 3 inputs → 2 hidden neurons → 1 output (binary classification)
Input: x = [0.5, 0.3, 0.8]ᵀ
Layer 1: W1 = [[0.4, 0.2, 0.6], b1 = [0.1, −0.1]ᵀ [0.3, 0.7, 0.1]] z1 = W1 · x + b1 = [0.4×0.5 + 0.2×0.3 + 0.6×0.8 + 0.1, 0.3×0.5 + 0.7×0.3 + 0.1×0.8 + (−0.1)] = [0.20 + 0.06 + 0.48 + 0.10, 0.15 + 0.21 + 0.08 − 0.10] = [0.84, 0.34] a1 = ReLU(z1) = [max(0,0.84), max(0,0.34)] = [0.84, 0.34]
Layer 2: W2 = [[0.5, 0.9]], b2 = [−0.2] z2 = W2 · a1 + b2 = 0.5×0.84 + 0.9×0.34 + (−0.2) = 0.42 + 0.306 − 0.20 = 0.526 ŷ = sigmoid(0.526) = 1/(1+e^−0.526) ≈ 0.629
Result: Model predicts 62.9% probability of class 1. With default threshold 0.5 → predicted class = 1

Weight Matrix Dimensions

📐 How to Figure Out Weight Matrix Shapes

Weight matrix W between layer with n neurons and layer with m neurons:

W shape = [m × n] (output_neurons × input_neurons)

Example: 3 inputs → 4 hidden → 2 outputs:

→ W1: [4 × 3]   b1: [4 × 1]

→ W2: [2 × 4]   b2: [2 × 1]

Forward Propagation in Detail

Forward propagation is the process of passing an input through a neural network layer by layer to produce an output prediction. It's the "forward pass" — information flows in one direction: input → hidden → output.

Figure 12.1 — Forward Propagation Data Flow

At each layer, the same two operations repeat:

  1. Weighted Sum: z = W · input + b (matrix multiplication plus bias)
  2. Activation: a = f(z) — apply the non-linear activation function

The outputs of each layer are stored (cached) during the forward pass because they are needed for backpropagation. This is why deep networks require significant memory.

💡 The Hidden Layer's Role

Each hidden layer is progressively learning more abstract features. In image classification: Layer 1 might learn edges → Layer 2 learns textures → Layer 3 learns shapes → Layer 4 learns objects. This hierarchical feature learning is what makes deep networks so powerful.

13

Backpropagation

Backpropagation is the algorithm that makes deep learning possible. It solves a fundamental problem: how do we know how much each hidden neuron contributed to the final error?

The Core Problem

For the output layer, it's easy — we compute the error directly: (prediction − truth). But for hidden layers, there are no target values. We can't directly measure how "wrong" a hidden neuron was.

📌 Backpropagation's Answer

Propagate the output error backwards through the network using the chain rule of calculus. Each hidden neuron receives a "blame" proportional to how much it contributed to the output error.

The Chain Rule — Core Concept

The chain rule says: if A affects B, and B affects C, then A's effect on C can be computed as (A's effect on B) × (B's effect on C).

∂L/∂W₁ = (∂L/∂ŷ) · (∂ŷ/∂z₂) · (∂z₂/∂a₁) · (∂a₁/∂z₁) · (∂z₁/∂W₁) Chain rule: how much does changing W₁ affect the final loss L?
Figure 13.1 — Forward and Backward Pass: Gradient flows opposite to data

Complete Backpropagation Worked Example

📐 Full Forward + Backward Pass — Step by Step
Network: 2 inputs → 1 hidden neuron → 1 output neuron Activation: Sigmoid everywhere. Loss: MSE = (y − ŷ)² x = [1.0, 0.5], y = 1 (true label), α = 0.1
Init Weights: w₁=0.4, w₂=0.6 (input→hidden), b₁=0.1 w₃=0.7 (hidden→output), b₂=0.2
━━ FORWARD ━━ ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Hidden z₁: z₁ = (1.0×0.4) + (0.5×0.6) + 0.1 = 0.40 + 0.30 + 0.10 = 0.80
Hidden a₁: a₁ = σ(0.80) = 1/(1+e^−0.8) = 1/1.449 ≈ 0.690
Output z₂: z₂ = (0.690 × 0.7) + 0.2 = 0.483 + 0.20 = 0.683
Output ŷ: ŷ = σ(0.683) = 1/(1+e^−0.683) ≈ 0.664
Loss L: L = (y − ŷ)² = (1 − 0.664)² = (0.336)² ≈ 0.1129
━━ BACKWARD ━━ ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
∂L/∂ŷ: ∂L/∂ŷ = −2(y − ŷ) = −2 × 0.336 = −0.672 (for MSE: derivative of (y-ŷ)² w.r.t. ŷ)
∂ŷ/∂z₂: σ'(z₂) = σ(z₂)(1−σ(z₂)) = 0.664 × 0.336 ≈ 0.2231
δ_output: δ₂ = ∂L/∂ŷ × ∂ŷ/∂z₂ = (−0.672) × 0.2231 ≈ −0.1499
Update w₃: ΔW₃ = −α × δ₂ × a₁ = −0.1 × (−0.1499) × 0.690 ≈ +0.01034 w₃_new = 0.7 + 0.01034 = 0.7103
δ_hidden: σ'(z₁) = σ(z₁)(1−σ(z₁)) = 0.690 × 0.310 = 0.2139 δ₁ = δ₂ × w₃ × σ'(z₁) = (−0.1499) × 0.7 × 0.2139 ≈ −0.02244
Update w₁,w₂: Δw₁ = −α × δ₁ × x₁ = −0.1 × (−0.02244) × 1.0 ≈ +0.002244 Δw₂ = −α × δ₁ × x₂ = −0.1 × (−0.02244) × 0.5 ≈ +0.001122 w₁_new = 0.4 + 0.002244 = 0.4022 w₂_new = 0.6 + 0.001122 = 0.6011

Gradient Descent

The core optimization algorithm. We compute the gradient (slope/derivative) of the loss with respect to each weight, then update weights in the opposite direction (downhill on the loss surface).

Wnew = Wold − α · ∇L(W) Move opposite to the gradient (downhill). α controls step size.
Figure 14.1 — Gradient Descent: Moving downhill on the loss surface

Too small (e.g., 0.0001)Tiny steps toward minimumExtremely slow convergence; may get stuck in local minima

Advanced Optimizers

Momentum

Adds a "velocity" term to gradient descent. Instead of jumping in the exact gradient direction every step, momentum keeps moving in the previous direction with some fraction of its prior velocity. This smooths oscillations and helps escape saddle points.

v = β·v_prev + (1−β)·∇L
W = W − α·v

Adaptive Learning Rate

The learning rate decreases as training progresses. Early on: large steps for fast learning. Later: small steps for fine-tuning near the optimum. Prevents overshooting when close to convergence.

Adam (Adaptive Moment Estimation)

Combines Momentum + Adaptive learning rate. Computes individual learning rates for each parameter. Industry standard — almost always the first optimizer to try. Fast and robust.

RMSProp

Adapts learning rate per-parameter based on recent gradient magnitudes. Especially effective for RNNs and problems with non-stationary gradients.

Weight Initialization

How you set the initial weights before training begins has a profound effect on whether the network learns at all. Poor initialization can cause the network to train extremely slowly, fail to converge, or learn nothing at all.

Why Initialization Matters

❌ Zero Initialization

If all weights start at 0, every neuron in a layer computes the exact same output and the exact same gradient. They all update in the exact same way forever. The layer is "stuck" — it can never learn different features. This is called the symmetry problem.

❌ Large Initial Weights

Very large weights create strong pre-fitted connections. The network starts with an "opinion" that is hard to change. Gradients may explode or saturate. Essentially, the network is already "committed" before training begins.

✅ Small Random Weights

Break symmetry (every neuron is different), but be careful about scale. Too small → vanishing gradient. Just right → efficient learning. The goal is to keep activations and gradients in a reasonable range throughout the network.

✅ Activation-Aware Init

Modern methods choose the weight scale based on the activation function used. Xavier for Sigmoid/Tanh. He/Kaiming for ReLU. This accounts for the mathematical properties of each activation's derivative.

Initialization Methods

MethodFormulaBest For
Uniform Random W ~ Uniform(−0.01, +0.01) Simple baseline. Breaks symmetry.
Xavier / Glorot W ~ N(0, 1/n_in) or Uniform(±√(1/n_in)) Sigmoid and Tanh activations. Maintains variance across layers.
He / Kaiming W ~ N(0, 2/n_in) ReLU activations. Factor of 2 compensates for ReLU zeroing half its inputs.
One-based W = 1 (all ones) Rarely used. Similar to zero init — symmetry problem persists.
📐 Xavier vs He — Which to Pick?

Layer has n_in = 4 input neurons:

Xavier: std = √(1/4) = 0.5 → W ~ N(0, 0.25)

He: std = √(2/4) = 0.707 → W ~ N(0, 0.5)

He initialises with larger values because ReLU loses half its range (negatives become 0), so the variance needs to be larger to compensate and keep signal strength through many layers.

Recurrent Neural Networks (RNN)

Standard feed-forward networks assume each input is independent. But in language, time series, video, and medical records, the order of data carries meaning. RNNs are designed to handle exactly this — sequential, time-dependent data.

Feed-Forward Network

Each input is processed independently. No memory of previous inputs. Suitable for: tabular data, single images, static features.

x → f(x) → ŷ

Recurrent Network

Each input is processed WITH memory of all previous inputs. Hidden state carries information across time steps. Suitable for: text, audio, time series, video.

x_t, h_{t-1} → f → h_t, ŷ_t

Types of Sequence Tasks

Task TypeInput → OutputExample
Sequence ClassificationFull sequence → single labelSentiment analysis, video activity recognition, patient diagnosis
Next-Step PredictionSequence → next elementLanguage modeling (GPT), stock price prediction
Encoder-DecoderInput sequence → output sequenceMachine translation, text summarisation
Seq-to-Seq labelingEach timestep → labelNamed entity recognition, speech recognition

Backpropagation Through Time (BPTT)

Training an RNN requires "unrolling" the network across all time steps and then applying standard backpropagation. The key difference: weights are shared across time, so gradients from all time steps are summed to produce the final weight update.

⚠️ Vanishing Gradient in RNNs

As gradients flow back through many time steps, they get multiplied by the same weight matrix repeatedly. If this matrix has values < 1, the gradient shrinks exponentially and becomes effectively zero — the network can no longer learn long-range dependencies. This is the vanishing gradient problem. For very long sequences, Truncated BPTT limits backpropagation to a fixed window.

Evolution Beyond Basic RNN

1
Basic RNN

Handles short sequences. Suffers from vanishing gradient for long sequences. Memory loss over long distances.

2
LSTM (Long Short-Term Memory)

Introduces gates (input gate, forget gate, output gate) that explicitly control what information to keep, forget, and output. Can maintain relevant information for hundreds of time steps. Solved the vanishing gradient problem for most practical cases.

3
LSTM + Attention

Adds an "attention mechanism" that allows the model to selectively focus on specific past timesteps when generating each output, rather than relying solely on the compressed hidden state.

4
Transformers (Self-Attention) — Current Standard

Process ALL time steps in parallel (not sequentially). Every position directly attends to every other position. No recurrence at all. Scales much better with sequence length and GPU parallelism. Powers GPT, BERT, and virtually all modern NLP models.

Evolutionary Computing

Evolutionary computing is a family of heuristic optimization algorithms inspired by biological natural selection. Unlike gradient descent, it does NOT require the objective function to be differentiable — making it applicable to discontinuous, non-smooth, or black-box problems.

Biological Analogy

Natural Selection

Only individuals well-adapted to their environment survive to reproduce. "Fitness" determines survival.

Inheritance

Offspring inherit traits from parents — but not identically. Variation is introduced.

Mutation

Random changes introduce new traits, preventing the population from stagnating at local optima.

Exploration vs Exploitation

The fundamental challenge in any optimization is finding the right balance between:

Exploitation

Use what you already know works. Refine the current best solutions. Fast progress but risks getting trapped in local optima — you'll never discover if there's something better elsewhere.

Exploration

Try new, unknown solutions. Costly and may not always work. But essential for finding the global optimum and not just a local one. This is what mutation provides.

💡 Best Strategy

Start with high exploration (lots of diversity, many mutations) early in the search. As promising regions are found, gradually shift toward high exploitation (refine the best solutions). This mirrors how human problem-solving often works: broad brainstorming first, then focused refinement.

Conceptual Questions — Practice These

  1. Why can't an SLP solve XOR? — It can only create a linear decision boundary. XOR requires a non-linear boundary (two separate regions) which is impossible with a single straight line.
  2. What is the vanishing gradient problem? — When gradients shrink exponentially as they flow backward through many layers (or timesteps), making weights in early layers barely update.
  3. Why is accuracy misleading for imbalanced datasets? — A model predicting only the majority class can achieve high accuracy while completely failing on the minority class.
  4. What is the role of the bias term? — Shifts the activation function, allowing the neuron to fire at values other than zero. Like the intercept c in y=mx+c.
  5. Why can't we initialize all weights to zero? — All neurons in a layer would compute identical gradients and learn the same features. Symmetry must be broken.
  6. What is the difference between a parameter and a hyperparameter? — Parameters (weights, biases) are learned during training. Hyperparameters (learning rate, layers, batch size) are set before training.
  7. What does generalization mean? — The model's ability to perform well on data it hasn't seen during training, not just on the training data.
  8. Why do we need non-linear activation functions? — Without them, a deep network collapses to a single linear transformation, regardless of depth. Non-linearity enables complex decision boundaries.
  9. Compare BPTT and Truncated BPTT. — BPTT unrolls the full sequence; Truncated BPTT limits backpropagation to a fixed window to prevent vanishing gradients and reduce computation.
  10. Why did Transformers replace RNNs? — Transformers process all timesteps in parallel (no sequential dependency), scale better, and use self-attention to directly model any-distance dependencies without gradient decay.

Common Exam Mistakes

⚠️ Avoid These Mistakes

1. Forgetting the bias term in forward propagation calculations — always add b.

2. Using the wrong activation for the output layer — Sigmoid for binary, Softmax for multi-class, Linear for regression.

3. Confusing Precision and Recall — re-derive from CM every time: Precision=TP/(TP+FP), Recall=TP/(TP+FN).

4. Applying weight updates during testing — NEVER. Weights freeze after training.

5. Saying higher depth always helps — more layers → overfitting if not regularised properly.

6. Forgetting that Step/Sign functions cannot be used with backprop — they are not differentiable.

7. Treating accuracy as the primary metric on imbalanced data — use F1, Precision, Recall, or AUC instead.
